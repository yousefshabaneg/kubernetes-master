Lab Setup: MySQL StatefulSet in KinD

3. Create a Headless Service

Since we are deploying a MySQL StatefulSet, we need a Headless Service that allows each MySQL pod to have its own stable network identity.

Create a file called mysql-headless-service.yaml with the following content:

apiVersion: v1
kind: Service
metadata:
  name: mysql
  labels:
    app: mysql
spec:
  ports:
    - port: 3306
  clusterIP: None
  selector:
    app: mysql

Apply the service:

kubectl apply -f mysql-headless-service.yaml

4. Create a MySQL ConfigMap

MySQL requires a default configuration for the root password and database initialization. Create a ConfigMap to store these values.

Create a file called mysql-configmap.yaml:

apiVersion: v1
kind: ConfigMap
metadata:
  name: mysql-config
data:
  init.sql: |
    CREATE DATABASE IF NOT EXISTS mydb;
    CREATE USER 'user'@'%' IDENTIFIED BY 'password';
    GRANT ALL PRIVILEGES ON mydb.* TO 'user'@'%';
    FLUSH PRIVILEGES;

Apply the ConfigMap:

kubectl apply -f mysql-configmap.yaml

5. Create a MySQL StatefulSet

Now, let’s define the StatefulSet for MySQL, which includes three MySQL instances (replicas). Each pod will have its own persistent volume, dynamically provisioned using the standard storage class.

Create a file called mysql-statefulset.yaml:

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mysql
spec:
  serviceName: "mysql"
  replicas: 3
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - name: mysql
        image: mysql:5.7
        ports:
        - containerPort: 3306
        env:
        - name: MYSQL_ROOT_PASSWORD
          value: "rootpassword"
        volumeMounts:
        - name: mysql-data
          mountPath: /var/lib/mysql
        - name: mysql-init
          mountPath: /docker-entrypoint-initdb.d
      volumes:
      - name: mysql-init
        configMap:
          name: mysql-config
  volumeClaimTemplates:
  - metadata:
      name: mysql-data
    spec:
      accessModes: [ "ReadWriteOnce" ]
      resources:
        requests:
          storage: 5Gi
      storageClassName: standard

In this file:

  • The headless service (mysql) is referenced.

  • Three replicas of MySQL are deployed.

  • Each pod has its own PVC (based on the VolumeClaimTemplates), ensuring each pod’s data is stored separately.

  • The configMap is mounted to initialize the MySQL database with custom SQL.

Apply the StatefulSet:

kubectl apply -f mysql-statefulset.yaml

6. Check the StatefulSet and Pods

Once the StatefulSet is applied, check the status of your pods:

kubectl get pods # several times till all the pods are created

Each pod will have a name like mysql-0, mysql-1, and mysql-2. These are deployed in sequential order, and each pod will have its own persistent storage.

You can also verify that the PVCs have been created and bound to PVs:

kubectl get pvc

You should see output similar to:

xxxxxxxxxx

7. Accessing MySQL

To interact with your MySQL cluster, you can enter one of the MySQL pods:

xxxxxxxxxx
kubectl exec -it mysql-0 -- mysql -u root -p

Use the root password (rootpassword), and once inside, you can verify the database and user:

xxxxxxxxxx
SHOW DATABASES;
USE mydb;
SHOW TABLES;

You should see that the database mydb has been created as per the init.sql defined in the ConfigMap.

Try to login with the account that we created and create a table in the mydb database:

xxxxxxxxxx
kubectl exec -it mysql-0 -- mysql -u user -ppassword
xxxxxxxxxx
USE mydb;
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
SELECT * FROM users;
